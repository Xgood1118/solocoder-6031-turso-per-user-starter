在 Todos 列表的基础上扩展任务分组能力，新增 Projects 实体：每个 Project 有 name 和 description 字段，可以归属多个 Todo，Todo 表里加 project_id 外键（迁移时已有 Todo 的 project_id 设为 null 并显示"未分组"）。新建 Project 时允许选一个 emoji 作为图标，前端列表用 emoji 加上 name 一起展示；Project 删除时弹确认框，提示"该项目下还有 N 个未完成任务，是否继续"，确认后这些 Todo 的 project_id 置 null 而不是 cascade delete。

在 Todo 模型上扩展优先级和截止日期：priority 取值 low 或者 medium 或者 high（默认 medium），due_date 允许 null 也允许 ISO 字符串。前端 Todo 行左侧显示优先级色块，low 灰、medium 黄、high 红，按 due_date 升序排列时已过期且未完成的 Todo 飘红并置顶。Dashboard 顶部新增一个"今日到期"小卡片，统计 due_date 是今天且 completed 是 false 的 Todo 数量，点击跳到过滤后的列表。

在 Welcome 页面下新增 onboarding 引导流程：新用户首次进入时检测该用户的数据库是否已就绪（表 todos 是否存在），如果还没有就展示三步引导——选数据库区域（US 或者 EU 二选一）、选初始示例 Todo 模板（空白、工作、学习三选一）、最后确认创建并执行 migrations 脚本。引导过程每一步骤都有上一步和下一步按钮，步骤之间用进度条连起来，刷新页面后引导状态写入 localStorage 可以续上。
